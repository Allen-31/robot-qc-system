# 质检系统服务器部署指南

部署到服务器时，需要准备以下组件，并保证**后端能连上 Keycloak**。

---

## 一、要部署的东西

| 组件 | 说明 | 常见方式 |
|------|------|----------|
| **PostgreSQL** | 业务库（用户/角色权限等仍部分用本地表） | Docker / 云 RDS / 自装 |
| **Redis** | 缓存（若用到） | Docker / 云 Redis / 自装 |
| **Keycloak** | 认证：登录、用户、角色 | **必须单独跑**，推荐 Docker |
| **后端 (RobotQcSystem)** | Spring Boot jar，端口 8080 | `java -jar` / systemd / Docker |
| **前端** | 静态资源或 SPA | Nginx 托管 / 同机部署 |

---

## 二、部署顺序建议

1. **先起 PostgreSQL、Redis**（若用 Docker 可一起起）。
2. **再起 Keycloak**，并做**首次初始化**（创建 realm、client、测试用户）。
3. **起后端**，环境变量里填对 Keycloak 地址、数据库、Redis。
4. **起前端**（或 Nginx 指到前端构建产物）。

---

## 三、Keycloak 在服务器上的用法

### 1. 用 Docker 跑 Keycloak（推荐）

```bash
# 持久化数据卷，重启不丢
docker volume create keycloak_data

docker run -d --name keycloak \
  -p 8081:8080 \
  -v keycloak_data:/opt/keycloak/data \
  -e KEYCLOAK_ADMIN=admin \
  -e KEYCLOAK_ADMIN_PASSWORD=你的强密码 \
  quay.io/keycloak/keycloak start-dev
```

生产环境建议用 **`start`** 并配数据库，例如：

```bash
docker run -d --name keycloak \
  -p 8081:8080 \
  -v keycloak_data:/opt/keycloak/data \
  -e KEYCLOAK_ADMIN=admin \
  -e KEYCLOAK_ADMIN_PASSWORD=你的强密码 \
  -e KC_DB=postgres \
  -e KC_DB_URL=jdbc:postgresql://postgres主机:5432/keycloak \
  -e KC_DB_USERNAME=keycloak \
  -e KC_DB_PASSWORD=keycloak密码 \
  quay.io/keycloak/keycloak start
```

### 2. 首次初始化 realm（必做一次）

Keycloak 起来后，在**能访问到服务器 8081 的机器**上执行（或把脚本拷到服务器用 PowerShell / Bash 跑）：

- **Windows（PowerShell）**：  
  - 把 `scripts/keycloak-setup.ps1` 里的 `$KeycloakUrl` 改成 `http://你的服务器IP:8081`（或域名）。  
  - 设置密码：`$env:KEYCLOAK_ADMIN_PASSWORD = "Keycloak管理员密码"`  
  - 执行：`.\scripts\keycloak-setup.ps1`

- **Linux（可改用 curl）**：  
  用 Keycloak Admin API 按脚本逻辑：创建 realm `robot-qc`、client `robot-qc-frontend`（开启 Direct access grants）、角色 admin、用户 admin/admin123 等（参考脚本里的接口顺序）。

执行成功后，Keycloak 里才有 `robot-qc` realm 和登录用的 client、用户，后端才能登录成功。

## 四、后端需要配的环境变量

在运行 `java -jar` 或 Docker 跑后端时，建议用环境变量覆盖默认值：

| 变量 | 说明 | 示例 |
|------|------|------|
| `DB_URL` | 数据库 JDBC 地址 | `jdbc:postgresql://数据库主机:5432/robot_qc` |
| `DB_USERNAME` | 数据库用户名 | `postgres` |
| `DB_PASSWORD` | 数据库密码 | 强密码 |
| `KEYCLOAK_ISSUER_URI` | **Keycloak realm 地址** | `http://服务器IP:8081/realms/robot-qc` 或 `https://域名/realms/robot-qc` |
| `KEYCLOAK_ADMIN_USERNAME` | Keycloak 管理员账号（仅后端调 Admin API 用） | `admin` |
| `KEYCLOAK_ADMIN_PASSWORD` | Keycloak 管理员密码 | 与 Keycloak 启动时一致 |
| `KEYCLOAK_CLIENT_ID` | BFF 登录用 client | `robot-qc-frontend`（一般不改） |
| Redis | 若用 | `spring.data.redis.host` 等 |

**要点**：`KEYCLOAK_ISSUER_URI` 必须是**后端能访问的地址**。若后端和 Keycloak 在同一台机，用 `http://localhost:8081/realms/robot-qc` 或 `http://本机内网IP:8081/realms/robot-qc`；若 Keycloak 在另一台机，填那台机的 IP 或域名。

## 五、一键起「依赖」：Docker Compose 示例

项目里可放一个 `docker-compose.yml`，只负责 **PostgreSQL + Redis + Keycloak**，后端仍用 `java -jar` 或单独镜像起。

```yaml
# 使用方式：在服务器上 docker-compose up -d，再按上面「首次初始化」执行 keycloak-setup
version: "3"
services:
  postgres:
    image: postgres:15-alpine
    container_name: robot-qc-postgres
    environment:
      POSTGRES_DB: robot_qc
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: robot-qc-redis
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  keycloak:
    image: quay.io/keycloak/keycloak:latest
    container_name: robot-qc-keycloak
    command: start-dev
    environment:
      KEYCLOAK_ADMIN: admin
      KEYCLOAK_ADMIN_PASSWORD: admin
    ports:
      - "8081:8080"
    volumes:
      - keycloak_data:/opt/keycloak/data
    depends_on:
      postgres:
        condition: service_healthy

volumes:
  postgres_data:
  keycloak_data:
```

- 首次：`docker-compose up -d`，等 Keycloak 就绪后执行 **keycloak-setup**（Keycloak URL 指到本机 `http://服务器IP:8081`）。
- 后端启动时设置：  
  `DB_URL=jdbc:postgresql://localhost:5432/robot_qc`，`KEYCLOAK_ISSUER_URI=http://localhost:8081/realms/robot-qc`（本机）或 `http://服务器IP:8081/realms/robot-qc`（若后端不在本机）。

---

## 六、简要检查清单

- [ ] PostgreSQL 已启动，库 `robot_qc` 已建（或由后端 schema 初始化）。
- [ ] Redis 已启动（若项目用到）。
- [ ] Keycloak 已启动，**已执行 keycloak-setup（realm robot-qc、client、admin 用户）**。
- [ ] 后端环境变量中 `KEYCLOAK_ISSUER_URI`、`KEYCLOAK_ADMIN_*`、`DB_*` 正确。
- [ ] 前端请求的后端地址、登录接口（`/api/auth/login`）与后端一致；若前后端不同域，后端 CORS 已放行前端域名。

按上述步骤，Keycloak 在 Docker 里单独跑，后端通过环境变量指向它，即可在服务器上正常部署和登录。
